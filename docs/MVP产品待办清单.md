# AI 线上问题排查 MVP 产品待办清单

## MVP 目标

第一版只验证一件事：

> AI 是否能帮助研发和值班同学更快完成线上问题初筛，并形成一张可用于人工判断的证据卡片。

## 用户入口

用户进入页面后，只需要完成三步：

1. 输入 Jira 链接。
2. 补充排查条件。
3. 查看故障初筛卡片并反馈是否有帮助。

## 第一版必须完成

### 1. 输入条件

必须支持：

- Jira 链接
- 环境
- 时间窗口
- 问题类型
- 服务范围

建议支持：

- Trace ID
- 接口名
- 错误关键词

验收标准：

- 用户能明确告诉系统“查哪个问题、查哪个环境、查哪个时间段、查哪些服务”。
- 页面不要求用户理解复杂概念。

### 2. 初筛卡片

必须固定输出以下内容：

- 问题摘要
- 排查范围
- 影响信号
- 关键时间线
- 可能原因 Top 3
- 证据
- 下一步建议
- 信息缺口

验收标准：

- 用户可以直接把卡片内容发给研发或值班同学。
- 卡片能帮助用户快速决定下一步查什么。

### 3. 人工反馈

必须收集：

- 是否有帮助
- 是否命中根因
- 实际根因
- 备注

验收标准：

- 每次分析后都能沉淀一次人工判断。
- 后续可以用这些反馈评估 MVP 是否有效。

## 暂不做

- 不做自动告警触发
- 不做自动发现服务
- 不做自动扩圈
- 不做自动处置
- 不做代码修复
- 不做复盘报告
- 不做多轮对话

## 演示路径

演示时按以下路径讲：

1. 研发收到一个 Jira 问题。
2. 打开页面，输入 Jira 链接。
3. 选择环境、时间窗口、问题类型和服务范围。
4. 点击生成初筛卡片。
5. 查看 AI 给出的摘要、可能原因、证据和下一步建议。
6. 人工判断是否有帮助、是否命中根因。

## MVP 成功标准

第一阶段不追求 AI 直接给出最终根因，而是看：

- 是否减少人工跨平台查资料的时间
- 是否更快形成初步判断
- 是否能给出可追溯证据
- 是否让新人或值班同学更快进入排查状态
- 用户是否愿意继续使用

## 建议评估指标

- 初筛卡片生成耗时
- 用户主观节省时间
- 根因命中率
- 有帮助反馈比例
- 信息缺口出现频率
